| 法规 | 地区 | 适用性 | 触发条件 |
|------|------|--------|---------|
| **PIPL** | 中国 | ✅ 强制 | 中国公民个人数据出境 |
| **GDPR** | 欧盟 | ✅ 强制 | 欧盟用户数据出境 |

| **日本 APPI** | 日本 | ⚠️ 可能 | 未来日语支持 |

## 二、PolyTalk 数据流分析

### 2.1 当前架构

```
┌─────────────────────────────────────────────────────────────┐
│                       用户数据类型                            │
├─────────────────────────────────────────────────────────────┤
│ 1. 用户注册信息（email, username, age）                      │
│ 2. 学习记录（conversations, messages, progress）             │
│ 3. 语音数据（audio_url, STT/TTS 处理）                       │
│ 4. AI 对话内容（发送到 GLM/GPT/Claude）                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                       数据出境路径                            │
├─────────────────────────────────────────────────────────────┤
│ 路径 A：中国用户 → Oracle 云 → GLM API（智谱，中国）          │
│         ✅ 合规：数据在中国境内处理                           │
│                                                              │
│ 路径 B：中国用户 → Oracle 云 → GPT/Claude（美国）            │
│         ⚠️ 风险：违反 PIPL 数据本地化要求                     │
│                                                              │
│ 路径 C：欧盟用户 → Oracle 云 → GLM/GPT/Claude                │
│         ⚠️ 风险：需 GDPR 标准合同条款（SCC）                  │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键风险点

| 风险 | 描述 | 法规 |
|------|------|------|
| **故障转移出境** | GLM 故障时自动切换到 GPT/Claude | PIPL |
| **语音数据出境** | 用户语音发送到国外 AI 服务 | PIPL/GDPR |
| **对话内容存储** | AI 对话记录可能含敏感信息 | PIPL/GDPR |
| **缺乏用户同意** | 无明确的跨境传输同意 | PIPL/GDPR |

## 三、PIPL 合规要求（中国）

### 3.1 数据出境条件

根据《个人信息保护法》第 38-43 条，数据出境需满足：

| 条件 | 要求 | PolyTalk 状态 |
|------|------|--------------|
| **必要性** | 确需出境才能实现业务目的 | ⚠️ 需评估 |
| **安全评估** | 通过国家网信办安全评估 | ❌ 未完成 |
| **标准合同** | 与境外接收方签订 SCC | ❌ 未完成 |
| **用户同意** | 单独同意跨境传输 | ❌ 未实现 |
| **认证** | 通过个人信息保护认证 | ❌ 未完成 |

### 3.2 必须安全评估的情形

以下情形**必须**进行国家网信办安全评估：

- [ ] 处理 100 万人以上个人信息
- [ ] 累计向境外提供 10 万人以上个人信息
- [ ] 累计向境外提供 1 万人以上敏感个人信息

**PolyTalk 判断**：MVP 阶段用户量小，暂不触发强制评估，但仍需：
1. 签订标准合同
2. 获得用户单独同意
3. 进行个人信息保护影响评估（PIA）

### 3.3 建议措施

```typescript
// 后端：根据用户地区选择 AI 厂商
class AIProviderSelector {
  select(user: User): AIProvider {
    // 中国用户：仅使用 GLM，禁止故障转移到美国服务
    if (user.region === 'CN') {
      return AIProvider.GLM;
    }
    
    // 欧盟用户：优先 GLM，备选 GPT（需 SCC）
    if (user.region === 'EU') {
      return this.hasSCC() ? AIProvider.GPT : AIProvider.GLM;
    }
    
    // 其他地区：多厂商路由
    return this.router.select();
  }
}
```

## 四、GDPR 合规要求（欧盟）

### 4.1 跨境传输机制

| 机制 | 条件 | PolyTalk 适用性 |
|------|------|----------------|
| **充分性决定** | 目标国有欧盟认可 | ❌ 美国、中国均无 |
| **标准合同条款（SCC）** | 签署欧盟模板合同 | ⚠️ 需与 OpenAI/Anthropic 签订 |
| **约束性企业规则（BCR）** | 集团内部转移 | ❌ 不适用 |
| **用户明确同意** | 告知风险后同意 | ✅ 需实现 |

### 4.2 用户权利

GDPR 要求提供以下权利，PolyTalk 需实现：

| 权利 | API 实现 | 状态 |
|------|---------|------|
| 访问权 | `GET /users/me` | ⚠️ 待实现 |
| 删除权（被遗忘权） | `DELETE /users/me` | ⚠️ 待实现 |
| 携带权 | `GET /users/me/export` | ⚠️ 待实现 |
| 更正权 | `PATCH /users/me` | ⚠️ 待实现 |
| 反对权 | `POST /users/me/opt-out` | ⚠️ 待实现 |
