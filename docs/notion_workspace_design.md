# PolyTalk Notion 工作区设计

> **版本**: v1.0
> **日期**: 2026-03-03
> **适用项目**: PolyTalk
> **技术栈**: NestJS + Fastify + SQLite + Prisma
> **团队**: 司礼监、兵部、户部、工部、礼部、刑部、吏部（7 个 Agent）

---

## 1. 设计目标

PolyTalk Notion 工作区用于统一管理研发协作、冲刺执行、文档沉淀、会议决策与里程碑跟踪，确保 7 个 Agent 在同一套项目视图下协同推进。

工作区设计遵循以下原则：

- **单一事实来源**：任务、文档、会议、里程碑均可追溯
- **跨部协同**：明确部门职责、依赖与交接关系
- **面向交付**：所有页面围绕 MVP 路线和每日冲刺展开
- **轻量可执行**：字段少而关键，避免过度维护成本

## 3. 项目看板（Kanban）

### 3.1 数据库定位

**数据库名称**：`PolyTalk 项目看板`

**用途**：

- 管理从需求池到交付完成的全生命周期任务
- 作为各部门执行任务的主入口
- 与冲刺计划、里程碑、会议纪要建立关联

### 3.2 状态分组

按 `状态` 属性建立 Board 视图，固定分组如下：

| 状态 | 说明 |
|------|------|
| **Backlog** | 需求池，待评估 |
| **To Do** | 已确认，待分配或待启动 |
| **In Progress** | 执行中 |
| **Review** | 提交审查/测试/验收 |
| **Done** | 已完成并验收 |

### 3.3 字段设计

| 字段名 | 类型 | 示例 | 用途 |
|--------|------|------|------|
| 任务名称 | Title | 实现 JWT 登录接口 | 任务主标题 |
| 状态 | Select | In Progress | 看板列状态 |
| 部门 | Select | 工部 | 归属部门 |
| 负责人 | Select | 工部严世蕃 | 当前执行人 |
| 优先级 | Select | P1 | 任务优先级 |
| 截止日期 | Date | 2026-03-07 | 交付时限 |
| Sprint | Relation | Sprint D3 | 关联冲刺日计划 |
| 里程碑 | Relation | 用户认证 | 关联所属里程碑 |
| 依赖任务 | Relation | 初始化 Prisma Schema | 标记前置任务 |
| 风险等级 | Select | 中 | 便于司礼监预警 |
| 预计工时 | Number | 4 | 用于容量估算 |
| 实际工时 | Number | 5 | 用于复盘 |
| 交付物 | Text | `/backend/src/auth/*` | 输出内容说明 |
| 验收标准 | Text | 登录成功返回 access token | Review 标准 |
| 阻塞原因 | Text | 等待数据库 schema 确认 | 阻塞记录 |
| 更新时间 | Last edited time | 自动 | 最后变更时间 |

### 3.4 推荐视图

| 视图名 | 类型 | 筛选/分组 |
|--------|------|-----------|
| 全量看板 | Board | 按状态分组 |
| 本周待办 | Table | 截止日期在 7 天内且未完成 |
| 按部门 | Board | 按部门分组 |
| 高优先级 | Table | 优先级为 P0/P1 |
| 阻塞任务 | Table | 阻塞原因非空 |

### 3.5 属性选项建议

**部门枚举**：

- 🏛️ 司礼监：统筹、分派、协调、验收推进
- ⚔️ 兵部：技术执行、安全修复、运维支持
- 💰 户部：资源跟踪、数据统计、排期管理
- 🔧 工部：功能开发、架构搭建、工程实现
- 🎭 礼部：文档、沟通、用户体验、发布说明
- ⚖️ 刑部：审计、合规、问题追踪
- 📋 吏部：Agent 配置、权限管理、组织维护

**优先级枚举**：

- P0：紧急（安全漏洞、阻塞问题）
- P1：高优（核心功能、关键路径）
- P2：普通（常规任务）
- P3：低优（优化、非紧急）

## 9. 使用指南

### 9.1 日常使用流程

1. **朝会（09:00）**
   - 更新冲刺计划状态
   - 确认今日任务
   - 识别阻塞项

2. **任务执行（日间）**
   - 更新看板任务状态
   - 记录实际工时
   - 标记阻塞原因

3. **夕会（18:00）**
   - 汇报今日完成
   - 更新进度百分比
   - 调整明日计划

4. **文档更新（随时）**
   - 提交新文档到文档库
   - 更新版本号和变更摘要
   - 关联相关任务

### 9.2 通知机制

- **任务分配**：通过 `sessions_send` 通知目标部门
- **阻塞预警**：司礼监每日检查阻塞任务
- **里程碑风险**：进度低于 50% 自动预警

### 9.3 维护建议

- 每日更新任务状态
- 每周归档已完成任务
- 每次发布更新文档库
- 每个里程碑更新进度
